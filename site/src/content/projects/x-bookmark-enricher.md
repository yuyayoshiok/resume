# x-bookmark-enricher — Xのブックマークを「使える知識」に変えてメモ帳へ取り込む

> X（旧Twitter）でブックマークした投稿を自動で取得し、本文やリンク先記事を読み込んでAIが要約・タグ付け。その結果を自分のメモ帳（Obsidian Vault）に整理して保存するツール。「あとで読む」で埋もれがちなブックマークを、検索・活用できる知識ベースに変える。

- **保存先**: **Obsidian Vault（個人のメモ帳アプリ）の `04_bookmarks_x/` フォルダにインポート**

---

## 概要

このツールの一番の特徴は、**処理結果をすべてObsidian Vaultに保存（インポート）する**こと。X上でブックマークしただけでは情報が流れていくが、これを通すとメモ帳の中で「要約・タグ・カテゴリ付きの整理されたノート」として蓄積され、後から検索したり記事ネタに使えるようになる。

処理は3段階:

```
1. 取得    Xの投稿本文・リンク先記事を読み込む
2. AI付加  要約・タグ付け・カテゴリ分類・「記事ネタ度」のスコアリング
3. 保存    メモ帳(Obsidian)に整理された形で書き込む（インポート）
```

---

## 主な機能

### ブックマークの取り込みと整理
- 投稿本文・引用ツイート・リンク先記事の本文まで取得
- X記事（長文投稿）は専用の `04_bookmarks_x/記事/` フォルダへ自動で振り分け
- 同じ投稿の重複ノートは自動で削除

### AIによる「情報の格上げ」
ブックマーク1件ごとに、AIが次を生成してメモ帳に付与:
- 1行サマリー / 200-400字の詳細要約 / 重要ポイント
- タグ（3-5個）とカテゴリ（AI・プログラミング・ビジネスなど13分類）
- **記事ネタ度（1-10点）**: ブログ記事に発展させられそうかをAIが採点。高得点のものを後でまとめて記事化できる

### まとめ記事の自動生成
- **週次キュレーション**: 記事ネタ度の高いブックマークとニュース記事を集めて、週1回の記事下書きをAIが作成 → `02_note/` へ
- **テーマ別まとめ**: 「AI」など指定したテーマで関連ブックマークを集め、「みんなが言っていること」を統合した記事を生成
- **ダッシュボード**: メモ帳全体の統計（処理済み件数・カテゴリ分布・記事ネタ候補top20・興味の傾向）を表形式で自動生成

## 技術スタック

- **言語**: TypeScript（型で安全に書ける言語）/ Node.js
- **AI**:
  - Groq（高速・低コストなAI、`llama-3.3-70b`）をメインに
  - Google Gemini を予備として併用（Groqが使えないとき自動で切替）
- **投稿の取得**: FxTwitter API（認証不要の公開API・高速）＋ Playwright（ブラウザを自動操作して取得する予備手段）
- **メモ帳連携**: gray-matter（Markdownの情報欄を読み書き）でObsidian形式のノートを生成

## アーキテクチャ

```
        Obsidian Vault（メモ帳・iCloud同期）
        ├─ 04_bookmarks_x/        ← ブックマークのノート（ここに保存）
        │   └─ 記事/              ← X長文記事は自動でここへ
        ├─ 02_note/               ← 生成された記事下書き
        └─ 05_moc_bookmarks/      ← ダッシュボード（統計・索引）
                ▲
                │ ノートを読み書き
        ┌───────┴────────────────────────────┐
        │  x-bookmark-enricher（Node.js）     │
        │  ① parser   メモ帳のノートを読む    │
        │  ② scraper  X投稿・記事を取得       │
        │  ③ enricher AIで要約・タグ付け      │
        │  ④ writer   メモ帳に書き戻す        │
        └───────┬───────────────┬─────────────┘
                ▼               ▼
          FxTwitter API   Groq / Gemini（AI）
          + Playwright
```

## 技術的な見どころ

- **大量処理に強い並列設計**: 取得は3件・AI要約は10件を同時並行で処理。数千件のブックマークも現実的な時間でさばける。AIの利用上限に達したら自動で停止・再開
- **AIの二段構え**: メインのGroqが使えなくなったらGeminiへ自動で切り替え。どちらも失敗したらそのノートはスキップして処理を止めない
- **取得手段の多重化**: まず高速なAPIで試し、ダメならブラウザ自動操作（Playwright）に切り替え。リンク先の記事本文まで読み込んで要約の材料にする
- **メモ帳ネイティブな作り**: Obsidianの情報欄（frontmatter）を正しく生成し、特別なプラグインなしでも見られる表形式のダッシュボードを出力。既存ノートとリンクも張る
- **「記事ネタ度」スコアリング**: ただ保存するだけでなく、ブログ記事化の可能性をAIが点数化。インプット（ブックマーク）からアウトプット（記事）への橋渡しを自動化
