# youtube-importer — YouTube動画を自動で要約してメモ帳に取り込む

> 見たいYouTubeチャンネルを登録しておくと、新着動画を自動で検知し、内容を文字起こし＆AI要約して、自分のメモ帳（Obsidian）に「読めるノート」として保存してくれるツール。動画を「見る」前に要点が手に入る。

- **リポジトリ**: `youtube-importer`
- **保存先**: Obsidian Vault（個人のメモ帳アプリ）の `06_youtube/` フォルダ

---

## 概要

YouTubeの新着動画を1時間ごとにチェックし、見つけたら次の処理を全自動で行う:

```
新着動画を検知 → 音声を文字起こし → AIで要約・タグ付け
  → 読みやすいノートに整形 → メモ帳(Obsidian)へ保存 → Slackで通知
```

現在自己啓発系の9チャンネルを監視し、これまでに200本以上の動画を処理済み。
動画を実際に見なくても、要点・キーワード・印象的な発言が手元のメモ帳に貯まっていく。

---

## 主な機能

- **新着の自動検知**: 登録チャンネル（`channels.txt` に記載）の新着を、YouTubeの更新通知（RSS）で取得
- **文字起こし**: 動画の音声を文字に変換（後述の多段フォールバックで確実に）
- **AI要約**: 3行要約・詳細要約・キーワード・実践ポイント・印象的な引用などを構造化して抽出
- **関連ノートの自動リンク**: メモ帳内の既存ノートと内容が近いものをAIが探し、相互リンクを張る（知識がつながる）
- **保存と通知**: 整形したノートをGitHub経由でメモ帳に保存し、Slackに完了通知

---

## 技術スタック

- **言語**: Python 3.11
- **自動実行**: GitHub Actions（毎時0分に自動起動する仕組み）
- **AI / API**:
  - Groq（高速・低コストなAI）— 文字起こし(Whisper)と要約(Llama)
  - Google Gemini — Groqが使えないときの予備
  - YouTube字幕取得 / yt-dlp（動画ダウンロード）
- **保存先連携**: GitHub API（メモ帳リポジトリへ書き込み）、Slack通知

---

## アーキテクチャ

```
GitHub Actions（毎時0分に起動）
  ▼
monitor.py（全体の司令塔）
  ├─ 1. 新着検知       channels.txt の各チャンネルをチェック、既読と比較
  ├─ 2. 文字起こし     ↓ 多段フォールバック（下記）
  ├─ 3. AI要約         Groq/Gemini で構造化JSONに要約
  ├─ 4. 関連ノート検索  メモ帳の既存ノートから関連を探してリンク
  ├─ 5. メモ帳へ保存    GitHub経由でObsidianの06_youtube/へ
  └─ 6. 状態を記録      state.json に既読・処理履歴を保存
```

### 文字起こしの「多段フォールバック」（確実に文字を得る工夫）
上から順に試し、失敗したら次へ自動で切り替える:
1. **YouTube字幕**（あれば最速・無料）
2. **Gemini**（動画URLをそのままAIに渡す）
3. **Groq Whisper**（音声をダウンロードしてAIで文字起こし）
4. **Gemini（音声ファイル）**（最終手段）

### 状態管理（`state.json`）
- 既読動画ID（重複処理を防ぐ）／未処理キュー／日別処理数／チャンネルID対応表
- 1日の処理上限（5本）を超えた動画はキューに溜め、翌日以降に自動処理

---

## メモ帳（Obsidian）への保存形式

各動画は、メモ帳が認識できる形式（先頭にタイトル・タグ等の情報欄を持つMarkdown）で保存される:

- **情報欄**: タイトル、チャンネル名、公開日、AIが付けた5-10個のタグ
- **本文**: 3行要約 → 詳細要約 → キーワード → 実践的な学び → 印象的な引用 → 関連ノートへのリンク → 論理構造

---

## 技術的な見どころ

- **止まらない文字起こし**: 字幕なし・ダウンロード失敗など、どこかで失敗しても次の手段に自動で切り替える4段構えで、確実に内容を取得
- **コストと精度の両立**: 高速・無料枠の大きいGroqをメインに、不足時だけGeminiへ。AIに送るテキストは1.2万字に切り詰めてコストを節約
- **賢い状態管理**: 1日5本の上限を設けつつ、超過分はキューに溜めて翌日処理。チャンネルIDをキャッシュしてAPI呼び出しを削減
- **知識がつながるメモ帳**: 単に保存するだけでなく、既存ノートとの関連をAIが見つけて相互リンク。知識ベースとして育っていく
- **運用の透明性**: Slackで新着・完了・エラーを通知。処理履歴はGit上に自動記録され、後から追える
